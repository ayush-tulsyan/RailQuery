Scope of Database

Database simplifies the task of storing all necessary details related to a railway inquiry system: train no., train type, etc. Details like route of the train, city, time of travel, cost, facilities, etc. are also included.



Motivation and Related Work

Railways is widely gaining importance not only due to the vast amount of services it provides but also its humongous expansion rate. Thus, an application which enables users to query important details related to a railways is a must in present scenario. But then, one may raise the question that these type of applications already exist, so why a new one? Considering the fact that this is a course project, its main aim is to get students familiar with the various DBMS systems and their technicalities. DBMS which we used is a Graph Database which is one of the famous database used now-a-days to store graph based data. Doing this project definitely taught us the basics of Graph Databases and their queries.



Description

In our project, we have tried to offer some of the functionalities offered by websites like erail.in, indiarailinfo.com. We keep a database of all the available trains and use this data answer the queries of our users.

The main algorithms which we are using are Dijkstra's Algorithm and BFS-Traversal. Using the built-in implementation of these algorithms in our database(neo4j), we compute all queries.

Our system offers a simple UI for the users. Based on the user input of source and target station, our system generates a set of available trains that would take one from the source to the destination. Our system also generates a map of India, and shows different trains and their routes.

Along with building this application, we have also looked upon the implementation of Neo4j, about how data is stored in it internally, how queries are optimized, etc. We have also looked into the technicalities of D3.js and how it converts a json file to a visual form.



Source of Data

Data was collected using railway API. Data in different formats was available like information regarding route of a train, canceled trains, rescheduled trains, etc. and all of this information was used in testing the scalability of our application.
